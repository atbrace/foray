## Vision

> Create a more frictionless pipeline between providing Claude with photos and measurements of an object and receiving a 3D printable asset. Trace actual contours from photos, reconcile multiple angles into a composite object, and identify what measurements are needed.

## Path Analysis

### 1. photo-to-contour-pipeline (RESOLVED, round 3, 3 experiments)

**Arc:** rembg pipeline discovery (001) -> RETR_CCOMP interior detection on synthetic masks (006) -> real-photo hole-filling confirmation (011)

**Key Findings:**
- rembg (U2Net) reliably produces high-fidelity outer silhouettes (143 vertices at eps=1px) on both renders and real photos
- The originally planned Canny-then-findContours approach is fundamentally wrong — fragments silhouettes into disconnected edge pieces. Correct approach: `findContours(mask, RETR_EXTERNAL)` directly on rembg alpha channel
- rembg fills interior through-holes mechanistically (alpha~254) — this is a property of salient-object-detection models, not a statistical edge case. RETR_CCOMP finds zero structural interior contours in rembg output despite holes being visually present

**Evidence quality:** Strong. Exp 001 independently verified via vertex count comparison (rembg 143 vs GrabCut 31). Exp 011 independently verified via direct alpha-channel measurement on both rendered and real photos.

**Gaps:** Interior hole detection requires a separate method (brightness-thresholding within rembg foreground mask is proposed but untested). Real-photo robustness tested on only 1 real image.

**Resolution justified:** Partially. The path answered its primary question (rembg cannot preserve interior holes), but this is more "blocked at interior features" than "pipeline complete."

### 2. claude-vision-real-photo-accuracy (RESOLVED, round 7, 6 experiments + 1 crash)

**Arc:** Crash (002, timeout) -> API key blocker (007) -> OpenCV fallback on synthetic data (012) -> scale-bar calibration discovery (016) -> Condition B validated on synthetics (019) -> real-photo validation (023). Crash 021 (timeout on ruler tick detection).

**Key Findings:**
- Shape classification is excellent (100% on tested examples, both synthetic and real)
- Claude systematically underestimates scale bar pixel width by 19-48% — this is a visual prior, not random noise
- Condition B (injecting pre-measured ppmm) achieves <=25% dimensional error on real photos by anchoring, bypassing the visual estimate entirely
- Color segmentation (HSV) is more reliable than Hough circles for calibration on real photos with cluttered backgrounds
- Uncalibrated dimension inference collapses on real photos (144.6% error)
- Thickness unreliable from face-on photos (49.1% error regardless of calibration)

**Evidence quality:** Strong for calibration mechanism (multiple experiments converge). Single real-photo validation (exp 023, flying disc) — narrow scope.

**Gaps:** Only 1 real photo tested. Color segmentation requires color contrast; white/grey PLA untested. Thickness measurement needs different angle.

**Resolution justified:** Yes, but narrow — validated on colored objects with neutral backgrounds.

### 3. contour-to-cadquery-extrusion (OPEN — should be RESOLVED, 13 experiments)

**Arc:** Complete validation in 1 experiment (003) -> 12 consecutive EXHAUSTED signals (008-030)

**Key Findings:**
- `.polyline().close().extrude()` handles all profile types (convex, non-convex, with interior holes) on the first attempt
- Y-axis inversion is a silent correctness bug: `[(x, -y) for x, y in image_verts]` required
- Interior holes: `.faces(">Z").workplane().moveTo(x,y).polyline(inner_verts).close().cutBlind(-depth)`
- All results independently verified by trimesh against ground-truth STLs (exact bbox and volume match)

**Evidence quality:** Very strong. Single definitive experiment with independent verification.

**The exhaustion loop (#32):** Experiment 003 answered the question completely. The planner correctly signaled EXHAUSTED starting in round 2. But `apply_guardrails` requires 2+ non-failure experiments for RESOLVED, and EXHAUSTED doesn't count as SUCCESS/PARTIAL. The evaluator recommended RESOLVED every round, guardrails rejected it, the path stayed OPEN, and the cycle repeated 12 times. This consumed 40% of the entire budget.

**Resolution justified:** The path should have been RESOLVED after exp 008 at the latest.

### 4. multi-view-silhouette-reconciliation (RESOLVED, round 3, 2 experiments)

**Arc:** Orthographic silhouette intersection (004) -> perspective rectification via homography (009)

**Key Findings:**
- 3-view orthographic projection recovers bounding boxes at 0% error for prismatic shapes (1.67% for cylinders — voxelization artifact)
- 5/6 synthetic objects have topology features detectable in at least one view
- 4-corner homography rectification reduces dimension error from 4-309% (unrectified) to <4% at up to 45 deg tilt
- Side view silhouettes add almost no topology information for depth-symmetric objects; front + top carry all signal
- Ambiguity boundary: bounding box always recoverable; wall thickness, hollow interiors, blind hole depths always require user measurement

**Evidence quality:** Strong for mathematics (quantitative tables, ground-truth comparison). All synthetic — real photos with lens distortion untested. Exp 004 had no eval JSON (assessment failed).

**Gaps:** Real-photo validation. Corner localization accuracy. Multi-face reconciliation (front + side -> 3D bbox) untested as an integrated pipeline.

**Resolution justified:** Justified for the sub-question of synthetic single-face geometry. Broader pipeline integration remains undemonstrated.

### 5. measurement-gap-identification (RESOLVED, round 4, 4 experiments)

**Arc:** CLI lacks image input (005, PARTIAL) -> API key blocker (010, FAILED) -> text-only baseline (015, 53% completeness) -> prompt rules fix (018, 100% completeness)

**Key Findings:**
- Text-only interview baseline: 53% completeness, 51% minimality across 5 benchmark objects
- Failure concentrated in snap-fit/tolerance parameters (assumed "standard") and user-preference inputs that don't fit M/V binary
- Three targeted prompt rules raised completeness from 0% to 100% on previously-failing objects: (1) snap-fit probe, (2) tolerance-critical mating, (3) user_inputs category
- Prompt engineering dominates — vision not required for measurement gap identification
- Minimality dropped to 22-58% (over-generates must_measure items), acceptable for interview use

**Evidence quality:** Medium. Self-evaluated against pre-defined ground truth in all cases. Concrete (items either appear or don't), but no independent third-party scoring.

**Gaps:** Vision's actual performance never tested (both attempts failed on environment, not methodology). Generalization beyond 5 benchmark objects unknown.

**Resolution justified:** Yes for the practical goal. The original question (does Vision outperform text?) was never answered — the path pivoted when prompt engineering solved the problem.

## Foray System Issues

### Bugs

1. **Exhaustion loop (#32, P1):** `apply_guardrails` has no EXHAUSTED-aware logic. When evaluator's RESOLVED recommendation is rejected, the path falls back to OPEN instead of INCONCLUSIVE. EXHAUSTED is a non-failure status, so it also doesn't trigger the consecutive failure circuit breaker. **Impact:** 40% of budget wasted.

2. **Single-experiment resolution blocked by guardrails:** The 2+ non-failure requirement prevented resolution of a path definitively answered by 1 experiment. Consider: if 3+ consecutive EXHAUSTED signals unanimously agree on resolution, override the minimum.

3. **Assessment failures (exps 004, 016):** Evaluator produced no eval JSON despite successful experiments. Finding summary defaults to "(assessment failed)." Exp 004's missing evaluation potentially delayed resolution of the multi-view path.

### Workflow Issues

4. **Executor should fail fast on missing credentials:** Experiments 007 and 010 failed due to missing API keys. The executor correctly stopped early, but the evaluator kept recommending "next experiment should resolve the API key" instead of recognizing this as an environment constraint. The fix is evaluator-side: recognize repeated credential failures and flag the path accordingly.

5. **No path-level BLOCKED for infrastructure issues:** When multiple experiments fail on the same environment constraint, the evaluator has no way to mark the path as infrastructure-blocked vs. hypothesis-blocked.

6. **No EXHAUSTED escalation mechanism:** After N consecutive unanimous EXHAUSTED signals, the system should auto-resolve or auto-mark-inconclusive without waiting for guardrails to approve.

### Prompt Issues

7. **Evaluator prompt needs closure rule:** "If EXHAUSTED signals are unanimous for 3+ consecutive rounds, recommend marking path complete."

8. **Evaluator should distinguish environment blockers from hypothesis failures:** Repeated environment failures should flag the path as "blocked pending infrastructure," not "ready for next experiment."
